Theory: Formalizing Value-at-Risk

VaR Definition

Value-at-Risk (VaR) is the loss threshold that will not be exceeded with \(\alpha\)% probability over a specified time horizon \(h\).

Mathematical Definition:

For confidence level \(\alpha \in (0,1)\) and loss random variable \(L\): \[\text{VaR}_\alpha = \inf\{l \in \mathbb{R} : P(L > l) \leq 1-\alpha\}\]

VaR is the smallest loss value \(l\) such that the probability of exceeding \(l\) is at most \((1-\alpha)\)

Equivalently, using the loss distribution’s quantile function: \[\text{VaR}_\alpha = F_L^{-1}(1-\alpha)\]

Where \(F_L^{-1}\) is the inverse cumulative distribution function (CDF), also called the quantile function.

VaR is the \((1-\alpha)\)-quantile of the loss distribution

Three essential components:

  1. Confidence level (\(\alpha\)): The probability that losses will not exceed VaR
  2. Time horizon (\(h\)): The period over which losses are measured
  3. Loss amount: The monetary value at risk

Interpretation:

  • “1-day 95% VaR = $X” means we expect to lose less than $X on 95% of days
  • On 5% of days (about once per month), we expect to lose more than $X.

Theory: Historical Simulation vs. Parametric Approach

Let’s calculate VaR using the historical simulation method.

Historical Simulation

Step 1: Sort all daily returns from worst to best

Step 2: Determine the position for your chosen confidence level \(\alpha\)%:

  • Number of exceptions allowed: \(n \times (1-\alpha)\)
  • Round conservatively to find the position \(k\)
  • The \(k\)-th worst observation is your VaR

Step 3: Convert the return to dollar loss for your portfolio value

Now let’s calculate VaR assuming returns follow a normal distribution.

Parametric Approach - Normal Distribution

Step 1: Calculate the distribution parameters from your data:

  • Sample mean: \(\mu = \frac{1}{n}\sum_{i=1}^{n} r_i\)
  • Sample standard deviation: \(\sigma = \sqrt{\frac{1}{n-1}\sum_{i=1}^{n}(r_i - \mu)^2}\)

Step 2: Apply the parametric VaR formula using the appropriate z-score: \[\text{VaR}_\alpha = -(\mu + \sigma \times z_{1-\alpha})\]

where \(z_{1-\alpha}\) is the standard normal quantile

To find \(z_{1-\alpha}\), you can use the Excel function NORM.INV(1-α, 0, 1)

Theory: Time Scaling and the Square-Root Rule

Scaling VaR Across Time Horizons

The Square-Root Rule: Under specific assumptions, VaR scales with the square root of time:

\[\text{VaR}_{h\text{-day}} = \text{VaR}_{1\text{-day}} \times \sqrt{h}\]

Required assumptions:

  1. Returns are independent across days (no autocorrelation)
  2. Returns are identically distributed (stable volatility)
  3. No mean reversion or momentum

Theory: Coherent Risk Measure

Coherent Risk Measure Axioms

A risk measure \(R\) is called coherent if it satisfies four axioms:

  1. Subadditivity (Diversification should reduce risk) \[R(L_1 + L_2) \leq R(L_1) + R(L_2)\]

  2. Positive Homogeneity (Scaling positions scales risk proportionally) \[R(\lambda L) = \lambda R(L), \text{ for every } \lambda > 0\]

  3. Monotonicity (Bigger losses mean bigger risk) \[R(L_1) \leq R(L_2) \text{ if } L_1 \leq L_2\]

  4. Translation Invariance (Adding cash reduces risk by that amount) \[R(L + a) = R(L) - a\]

VaR violation: VaR can fail subadditivity, especially for:

  • Portfolios with fat-tailed distributions
  • Concentrated credit exposures
  • Non-linear derivatives positions

Key insight: VaR satisfies only 3 of the 4 axioms. This single failure means VaR is not “coherent” in the mathematical sense.

Case Study: JP Morgan’s London Whale (2012)

JP Morgan’s $6.2 billion loss illustrates how VaR model changes can mask growing risks.

Late 2011 - Original Model:

  • Historical simulation VaR using full dataset
  • Result: VaR = $95-132 million
  • Status: Approaching risk limits
  • CIO position size: ~$157 billion notional

January 2012 - New “Enhanced” Model:

  • Switched to parametric (Gaussian) VaR approach
  • Used shorter, calmer data window
  • “Simplified” correlation calculations
  • Result: VaR dropped to $54-67 million overnight
  • Status: Comfortably within limits again

The Key Changes:

Aspect Old Model New Model Impact
Method Historical simulation Parametric/Gaussian Assumes normality
Data period Multiple years Shorter, recent only Excluded crisis volatility
Weighting Time-weighted Equal-weighted Reduced recent volatility impact
Implementation Established system Excel spreadsheet Introduced formula errors

What Actually Happened:

  • The Excel error: Divided by sum instead of average of volatilities (understated risk by ~50%)
  • VaR breaches: The desk exceeded its VaR limit on 71 days in Q1 2012
  • Actual losses: Daily losses exceeded $200M multiple times (3x the reported VaR)
  • Model reversion: In April 2012, switched back to old model → VaR jumped to $129M

The Governance Failures:

  • Model change approved without proper validation
  • Risk committee not informed of methodology change
  • Breaches explained away as “technical” or “model issues”
  • Traders influenced risk measurement process

Key Takeaways

  • VaR is simple and intuitive but has critical limitations

  • Different stakeholders need different VaR parameters

  • VaR fails to capture:

    • Tail risk severity
    • Liquidity risk
    • Model risk
    • Correlation breakdowns

  • Real failures (London Whale, 2008 crisis) show VaR’s limitations
